===============================

Q: What is Git?
A: Git is a distributed version control system to track changes in code.

Q: Difference between Git and GitHub?
A: Git = VCS tool; GitHub = cloud hosting service for Git repositories.

Q: What is a commit?
A: A snapshot of staged changes, stored permanently in Git history.

Q: Difference between local and remote repository?
A: Local: on your machine. Remote: hosted on server (GitHub, GitLab, etc.).

===============================
2Ô∏è‚É£ BRANCHING & MERGING
===============================

Q: How to create and switch branches?
A: git checkout -b branch_name

Q: Difference between git merge and git rebase?
A: Merge ‚Üí combine branches, keeps history.
Rebase ‚Üí replay commits, cleaner linear history.

Q: Fast-forward merge?
A: Branch can be moved forward without creating a merge commit.

Q: Conflict resolution?
A: Happens when same lines changed in different branches. Resolve manually and commit.

===============================
3Ô∏è‚É£ STAGING & COMMITTING
===============================

Q: Difference between git add and git commit?
A: git add ‚Üí stage changes
git commit ‚Üí save staged changes in repo

Q: git commit -m vs git commit --amend?
A: -m ‚Üí new commit
--amend ‚Üí modify last commit message or content

Q: Difference between git reset, git revert, git checkout?
A: reset ‚Üí move HEAD, can unstage or delete commits
revert ‚Üí create commit that undoes previous commit
checkout ‚Üí switch branch or restore file

===============================
4Ô∏è‚É£ REMOTE REPOSITORIES
===============================

Q: git fetch vs git pull?
A: fetch ‚Üí only downloads changes
pull ‚Üí fetch + merge

Q: git push vs git push --force?
A: push ‚Üí normal push
force ‚Üí overwrite remote branch, risky on shared branches

Q: How to clone a repository?
A: git clone <repo-url>

===============================
5Ô∏è‚É£ STASHING & RECOVERY
===============================

Q: What is git stash?
A: Temporarily saves changes without committing.

Q: Recover deleted commits?
A: git reflog ‚Üí shows all HEAD movements
git checkout <commit> ‚Üí recover commit

Q: git clean?
A: Remove untracked files:
git clean -n ‚Üí preview
git clean -f ‚Üí delete

===============================
6Ô∏è‚É£ TAGGING & VERSIONING
===============================

Q: Difference between lightweight and annotated tags?
A: Lightweight ‚Üí just a name
Annotated ‚Üí stores metadata (author, date, message)

Q: How to push tags to remote?
A: git push origin <tag-name>

===============================
7Ô∏è‚É£ COLLISION & HISTORY MANAGEMENT
===============================

Q: What is a detached HEAD?
A: HEAD points directly to commit, not branch. Changes may be lost if not handled.

Q: Difference between squash and fixup commits?
A: squash ‚Üí combines commits with message
fixup ‚Üí combines commits, discards commit message

Q: Git bisect?
A: Binary search to find commit that introduced a bug.

===============================
8Ô∏è‚É£ COMMON INTERVIEW QUESTIONS
===============================

Q: How does Git handle conflicts?
A: Manual merge required; resolved conflicts must be staged & committed.

Q: Explain difference: Git vs SVN?
A: Git ‚Üí distributed, each dev has full history
SVN ‚Üí centralized, requires server connection

Q: When to use rebase over merge?
A: Clean linear history, avoid multiple merge commits in feature branch.

Q: How to undo pushed commit?
A: git revert <commit> ‚Üí safe
git reset + force push ‚Üí risky

Q: What is the difference between origin and upstream?
A: origin ‚Üí your fork/clone remote
upstream ‚Üí original repository (main source)
