# Java Collections â€“ When & Why to Use

### 1. Array
- When to use:
  - Fixed-size collection of elements.
  - Fast index-based access (O(1)).
  - Minimal memory overhead.
- Example topics: Storing constants, lookup tables, configuration values.

### 2. ArrayList
- When to use:
  - Dynamic-sized array.
  - Fast random access (O(1)).
  - Iteration order matters.
- Example topics: Lists of students, tasks, dynamic datasets.

### 3. LinkedList
- When to use:
  - Frequent insertions/deletions at ends or in the middle.
  - Sequential access preferred over random access.
  - Implementing queue (FIFO) or stack (LIFO) operations.
- Example topics: Task queues, undo-redo history, BFS traversal.

### 4. ArrayDeque
- When to use:
  - Fast stack (push/pop) or queue operations.
  - Resizable array, no null elements allowed.
  - Better performance than LinkedList for deque operations.
- Example topics: Task scheduling, BFS traversal, undo-redo stacks.

### 5. TreeSet
- When to use:
  - Store sorted, unique elements automatically.
  - Efficient O(log n) add/remove/search.
  - Iterate in ascending (sorted) order.
- Example topics: Sorted leaderboards, unique sorted IDs, ranking systems.

### 6. HashSet
- When to use:
  - Store unique elements only.
  - Fast add, remove, contains (O(1)).
  - Order does not matter.
- Example topics: Unique value collections, lookup sets, membership checks.

### 7. LinkedHashSet
- When to use:
  - Store unique elements while maintaining insertion order.
  - Fast add, remove, contains (O(1)).
- Example topics: Caches, preserving order while removing duplicates, ordered sets.

### 8. HashMap
- When to use:
  - Key-value pairs with fast access (O(1)).
  - No ordering required.
  - One null key and multiple null values allowed.
- Example topics: Lookup tables, caching, frequency counts, dictionary-like mappings.

### 9. TreeMap
- When to use:
  - Key-value pairs with keys sorted automatically (natural or comparator).
  - Efficient log(n) access, insert, delete.
  - Iteration order is consistent based on sorted keys.
- Example topics: Leaderboard ranking, scheduled events, ordered dictionaries.
